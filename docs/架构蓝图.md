# 玄枢项目架构蓝图

## 📌 项目概述

**玄枢（XuanShu）** 是一个本地部署的AI Agent系统，支持单机和多机协作模式。核心目标是让用户在不依赖云端大模型的情况下，通过本地模型完成各类任务。

- **定位**：开源、轻量级、可扩展的本地AI Agent框架
- **技术栈**：Python 3.8+, FastAPI, SQLite
- **支持模式**：单机模式、集群协作模式
- **项目路径**：Windows: `J:\XuanShu` / Linux: `/www/wwwroot/local-agent`

## 🧠 核心概念

### 1. UniversalAgent
主Agent类，负责：
- 加载和管理Skills
- 处理用户消息
- 管理对话历史
- 调用模型

### 2. Skills Registry
技能注册中心：
- 动态加载技能
- 统一调用接口
- OpenAI Function Calling格式输出

### 3. Model Providers
模型提供商抽象：
- Ollama (本地)
- OpenAI 兼容 API
- 自定义 API

### 4. Cluster Mode
集群协作模式：
- Manager: 房间创建者，任务调度者
- Worker: 任务执行者
- 房间: 协作单元

## 📊 配置层级

```
config.py (全局配置)
    ├── MODEL_NAME: 模型名称
    ├── WEB_HOST/WEB_PORT: Web服务配置
    ├── CLUSTER_ENABLED: 集群开关
    ├── CLUSTER_ROLE: 角色 (manager/worker)
    └── SECURITY_* : 安全配置

model_providers.py (模型配置)
    └── data/model_config.json (持久化存储)

launcher.py (运行时配置)
    └── 用户交互选择
```

## 🔄 数据流

### 单机对话流程
```
用户输入 → web_app /api/chat
    → conversation_manager.add_user_message()
    → agent._process_simple()
        → model_providers.call_model()
        → skills_registry.execute()
    → conversation_manager.add_assistant_message()
    → 返回响应
```

### 集群协作流程
```
Manager: 创建房间 → 等待Worker加入 → 分配任务 → 收集结果
Worker: 发现房间 → 加入房间 → 接收任务 → 执行并上报
```
