# 玄枢项目架构蓝图

## 📌 项目概述

**玄枢（XuanShu）** 是一个本地部署的AI Agent系统，支持单机和多机协作模式。核心目标是让用户在不依赖云端大模型的情况下，通过本地模型完成各类任务。

- **定位**：开源、轻量级、可扩展的本地AI Agent框架
- **技术栈**：Python 3.10+, FastAPI, SQLite (WAL)
- **支持模式**：单机模式、集群协作模式
- **项目路径**：Windows: `J:\XuanShu` / Linux: `/www/wwwroot/local-agent`
- **统一UDP发现端口**：50005（局域网房间发现）
- **集群TCP端口**：30001（Manager通信端口）
- **Web服务端口**：30000（主界面）
- **文档版本**：v5.5.3

## 🧠 核心概念

### 1. UniversalAgent
主Agent类，负责：
- 加载和管理Skills
- 处理用户消息
- 管理对话历史
- 调用模型

### 2. Skills Registry
技能注册中心：
- 动态加载技能
- 统一调用接口
- OpenAI Function Calling格式输出

### 3. Model Providers
模型提供商抽象：
- Ollama (本地)
- OpenAI 兼容 API
- 自定义 API
- ConfigManager 单例管理配置持久化

### 4. Cluster Mode
集群协作模式：
- Manager: 房间创建者，任务调度者
- Worker: 任务执行者
- 房间: 协作单元
- UDP 50005端口: 局域网房间广播与发现（统一端口）
- 独立状态标志: broadcasting + scanning（分离广播与扫描）

### 5. Capability Assessor
智能能力评估：
- 扩展模型排行榜（几乎所有主流模型）
- 智能最长子串匹配
- 参数量自动估算规则（70b=0.90, 32b=0.85...）
- 云端API默认0.85，本地未知模型0.70

## 📊 配置层级

```
config.py (全局配置)
    ├── MODEL_NAME: 模型名称
    ├── WEB_HOST/WEB_PORT: Web服务配置
    ├── UDP_DISCOVERY_PORT=50005: 局域网发现统一端口
    ├── CLUSTER_ENABLED: 集群开关
    ├── CLUSTER_ROLE: 角色 (manager/worker)
    └── SECURITY_* : 安全配置

model_providers.py (模型配置)
    └── data/model_config.json (持久化存储，单例ConfigManager)

launcher.py (运行时配置)
    └── 用户交互选择
```

## 🔄 数据流

### 单机对话流程
```
用户输入 → web_app /api/chat
    → conversation_manager.add_user_message()
    → agent._process_simple()
        → model_providers.call_model()
        → skills_registry.execute()
    → conversation_manager.add_assistant_message()
    → 返回响应
```

### 局域网房间创建+发现流程
```
用户创建房间 → web_app /api/rooms/create
    → ensure_cluster_initialized() 从config_manager正确读取当前模型
        → ClusterManager.create_room()
        → discovery.update_room_info(房主信息)
        → discovery.start_hosting() 启动UDP广播（端口50005）
        → 同时discovery.start_scanning() 监听其他主机广播

其他主机 → /api/rooms/list
    → 合并本地房间 + 所有UDP发现的远程房间
        → 展示在UI房间列表，房主模型信息完整可见
```

### 集群协作流程
```
Manager: 创建房间 → UDP广播房间信息 → 等待Worker加入 → 分配任务 → 向所有浏览器推送 enter_collab_mode 事件 → 全体自动跳转协作对话 → 收集结果
Worker: UDP扫描发现房间 → 点击加入 → 接收任务 → 执行并上报结果
```
