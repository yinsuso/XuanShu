## 📌 P1 - 高优先级

### [P1-001] 单机模式房间协作功能不完整
- **描述**: 创建房间后无法真正进行协作，缺少任务分配和成员交互
- **影响**: 协作模式无法实际使用
- **发现日期**: 2026-05-08
- **发现人**: Agent
- **状态**: 🟢 已修复，房间列表、创建、详情页面、协作专区均已完善
- **修复内容**:
  - 房间创建后自动进入详情页面
  - 添加协作专区（任务发送、日志显示）
  - 添加解散房间功能
  - 添加成员自动刷新
  - 添加任务错误反馈机制
- **相关文件**:
  - `web_app.py`
  - `evolution/cluster/connection.py`
  - `web/static/index.html`

### [P1-002] 端口占用导致启动失败
- **描述**: 端口30000被占用时启动失败，没有自动切换端口
- **影响**: 用户无法启动服务
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: `find_available_port` 函数已存在，但uvicorn调用方式有误，已修正
- **相关文件**:
  - `web_app.py`

### [P1-003] 命令行选择模型后Web页面不同步
- **描述**: 命令行选择的模型在Web页面不显示
- **影响**: 用户体验不一致
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: init函数中添加了loadModelList()调用
- **相关文件**:
  - `web/static/index.html`

### [P1-004] 房间列表获取失败
- **描述**: 获取房间列表API返回错误，无法显示房间列表
- **影响**: 用户无法查看和加入房间
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: ClusterManager初始化时缺少owner_name和owner_model属性
- **相关文件**:
  - `evolution/cluster/connection.py`

### [P1-005] WebSocket连接403错误
- **描述**: `/api/cluster/ws/updates` 路由返回403错误
- **影响**: 实时更新功能无法使用
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: 移除WebSocket路由对ClusterNode的依赖，修复路由前缀配置
- **相关文件**:
  - `evolution/cluster/cluster_api.py`
  - `web_app.py`

### [P1-006] 成员信息无法修改
- **描述**: 协作页面中修改花名和模型后保存失败
- **影响**: 成员无法自定义个人信息
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: update_member API支持多种成员ID匹配方式
- **相关文件**:
  - `web_app.py`

### [P1-007] JSON导出MessageRole序列化错误
- **描述**: 导出JSON格式时提示 "Object of type MessageRole is not JSON serializable"
- **影响**: 无法导出JSON格式对话记录
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: format_value函数添加枚举类型处理，支持MessageRole.name
- **相关文件**:
  - `web_app.py`

### [P1-008] 创建房间模型列表不显示云端API
- **描述**: 创建房间页面只显示Ollama本地模型，不显示云端API配置模型
- **影响**: 用户无法选择云端模型创建房间
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: loadRoomModels函数同时加载本地Ollama模型和云端配置模型
- **相关文件**:
  - `web/static/index.html`

### [P1-009] 重新回复功能无效
- **描述**: 对话页面重新回复按钮点击后无响应
- **影响**: 用户无法重新生成回复
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: resendMessage函数使用api()函数保持一致性
- **相关文件**:
  - `web/static/index.html`

## 💡 P3 - 低优先级（未来改进）

### [P1-010] discovery.py构造函数缺少host_port参数
- **描述**: ClusterDiscovery构造函数在初始化时没有host_port参数，但在connection.py中调用时传入了该参数
- **影响**: 初始化发现服务时会报错
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 在discovery.py的__init__中添加host_port参数
- **相关文件**: `evolution/cluster/discovery.py`

### [P1-011] 局域网房间发现缺少API端点
- **描述**: 局域网UDP广播房间发现功能有底层实现，但缺少对外REST API暴露给前端
- **影响**: 其他玄枢项目无法在页面上发现局域网中的房间
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 添加三个新端点: /api/discovery/start_scan, /api/discovery/rooms, /api/discovery/stop
- **相关文件**: `web_app.py`

### [P1-012] 能力评估默认返回0.5分
- **描述**: 云端API和本地Ollama模型大部分不在硬编码排行榜里，直接返回默认0.5分
- **影响**: 用户所有模型都是0.5分，能力评估形同虚设
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 
  1. 大幅扩展模型排行榜，加入几乎所有主流模型
  2. 新增智能子串匹配（优先最长匹配）
  3. 新增基于参数量的自动估算规则（70b=0.90, 32b=0.85...）
  4. 云端API智能默认0.85，本地未知模型0.70
- **相关文件**: 
  - `evolution/cluster/capability.py`
  - `evolution/cluster/connection.py`

### [P3-001] 集群发现机制依赖局域网广播
- **描述**: 当前使用UDP广播发现节点，不支持跨网络
- **改进方向**: 支持手动指定节点地址
- **状态**: 🟢 已部分修复，新增扫描API
- **相关文件**: `evolution/cluster/discovery.py`

### [P3-002] 向量记忆功能需要手动启用
- **描述**: `USE_VECTOR_MEMORY` 默认关闭，需要手动安装依赖
- **改进方向**: 自动化检测和配置
- **状态**: 🔴 待处理
- **相关文件**: `config.py`, `memory_enhanced.py`
