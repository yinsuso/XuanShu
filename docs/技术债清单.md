## 📌 P1 - 高优先级

### [P1-001] 单机模式房间协作功能不完整
- **描述**: 创建房间后无法真正进行协作，缺少任务分配和成员交互
- **影响**: 协作模式无法实际使用
- **发现日期**: 2026-05-08
- **发现人**: Agent
- **状态**: 🟢 已修复，房间列表、创建、详情页面、协作专区均已完善
- **修复内容**:
  - 房间创建后自动进入详情页面
  - 添加协作专区（任务发送、日志显示）
  - 添加解散房间功能
  - 添加成员自动刷新
  - 添加任务错误反馈机制
- **相关文件**:
  - `web_app.py`
  - `evolution/cluster/connection.py`
  - `web/static/index.html`

### [P1-002] 端口占用导致启动失败
- **描述**: 端口30000被占用时启动失败，没有自动切换端口
- **影响**: 用户无法启动服务

- **状态**: 🟢 已修复
- **修复内容**: `find_available_port` 函数已存在，但uvicorn调用方式有误，已修正
- **相关文件**:
  - `web_app.py`

### [P1-003] 命令行选择模型后Web页面不同步
- **描述**: 命令行选择的模型在Web页面不显示
- **影响**: 用户体验不一致
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: init函数中添加了loadModelList()调用
- **相关文件**:
  - `web/static/index.html`

### [P1-004] 房间列表获取失败
- **描述**: 获取房间列表API返回错误，无法显示房间列表
- **影响**: 用户无法查看和加入房间

- **状态**: 🟢 已修复
- **修复内容**: ClusterManager初始化时缺少owner_name和owner_model属性
- **相关文件**:
  - `evolution/cluster/connection.py`

### [P1-005] WebSocket连接403错误
- **描述**: `/api/cluster/ws/updates` 路由返回403错误
- **影响**: 实时更新功能无法使用

- **状态**: 🟢 已修复
- **修复内容**: 移除WebSocket路由对ClusterNode的依赖，修复路由前缀配置
- **相关文件**:
  - `evolution/cluster/cluster_api.py`
  - `web_app.py`

### [P1-006] 成员信息无法修改
- **描述**: 协作页面中修改花名和模型后保存失败
- **影响**: 成员无法自定义个人信息
- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: update_member API支持多种成员ID匹配方式
- **相关文件**:
  - `web_app.py`

### [P1-007] JSON导出MessageRole序列化错误

- **影响**: 无法导出JSON格式对话记录

- **状态**: 🟢 已修复
- **修复内容**: format_value函数添加枚举类型处理，支持MessageRole.name
- **相关文件**:
  - `web_app.py`

### [P1-008] 创建房间模型列表不显示云端API

- **发现日期**: 2026-05-08
- **状态**: 🟢 已修复
- **修复内容**: loadRoomModels函数同时加载本地Ollama模型和云端配置模型
- **相关文件**:
  - `web/static/index.html`

### [P1-009] 重新回复功能无效

- **影响**: 用户无法重新生成回复

- **状态**: 🟢 已修复
- **修复内容**: resendMessage函数使用api()函数保持一致性
- **相关文件**:
  - `web/static/index.html`

### [P1-013] 创建房间后房主模型显示错误
- **描述**: 创建房间后，我的信息中显示的模型不是从model_config.json或Ollama中配置的当前模型，而是硬编码默认值
- **影响**: 用户明明切换了模型，但是房间里显示的模型不对
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 单机初始化集群节点时，从model_providers.config_manager单例中读取current_config的model_name，而不是硬编码
- **相关文件**:
  - `web_app.py`

### [P1-014] 局域网跨主机房间发现功能失效
- **描述**: 同一局域网（同一路由器/同一网段）内不同主机（一个Windows一个Linux）互相发现不了对方的房间，端口30000/30001/30002都开放了
- **影响**: 多机局域网协作模式无法正常工作，只能单机演示
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 
  1. 定义全局UDP_DISCOVERY_PORT=50005，所有节点统一频道
  2. 把单一running拆分成broadcasting和scanning独立状态标志
  3. 新增update_room_info方法，动态更新广播内容
  4. 创建房间后自动启动房主UDP广播
- **相关文件**:
  - `evolution/cluster/discovery.py`
  - `web_app.py`

### [P1-015] WebSocket端点参数错误导致连接断开
- **描述**: 房主端Web端连接时报错 TypeError: ws_updates() missing 1 required positional argument: 'request'
- **影响**: WebSocket实时更新功能无法使用
- **发现日期**: 2026-05-11
- **状态**: 🟢 已修复
- **修复内容**: WebSocket端点移除多余的 request 参数，改为从 websocket.scope 获取 app 实例
- **相关文件**: `evolution/cluster/cluster_api.py`

### [P1-016] 房间密码验证完全缺失
- **描述**: 设置了密码的房间，任何人不输入正确密码都可以直接加入
- **影响**: 加密房间形同虚设，安全无保障
- **发现日期**: 2026-05-11
- **状态**: 🟢 已修复
- **修复内容**: 在 ClusterServer._handle_client 中添加 SHA256 密码哈希校验
- **相关文件**: `evolution/cluster/connection.py`

### [P1-017] 加入房间使用多个原生prompt用户体验差
- **描述**: 加入房间流程使用多个浏览器原生prompt弹窗，需要手动输入花名和模型名，无法自动读取本地配置
- **影响**: 用户体验差，操作繁琐
- **发现日期**: 2026-05-11
- **状态**: 🟢 已修复
- **修复内容**: 新增完整精美的加入房间表单弹窗，一次性收集花名、自动模型下拉选择、密码输入
- **相关文件**: `web/static/index.html`

### [P1-018] 所有成员状态显示离线/未连接
- **描述**: 加入房间后所有成员列表显示状态都是离线，WebSocket显示未连接
- **影响**: 用户无法知道成员在线状态
- **发现日期**: 2026-05-11
- **状态**: 🟢 已修复
- **修复内容**: 新节点加入后设置 node.status = "active"，确保状态正确
- **相关文件**: `evolution/cluster/connection.py`, `web_app.py`

### [P1-019] 进入协作房间后模型无法正确调用model_config.json中配置的模型
- **描述**: 切换模型后，cluster相关的房主模型、节点模型、成员信息、UDP广播信息没有同步更新，导致房间内信息使用的模型与实际配置不一致
- **影响**: 用户明明切换了模型，但房间里实际使用的模型不对
- **发现日期**: 2026-05-11
- **状态**: 🟢 已修复
- **修复内容**: 增强/api/switch_model端点，新增全链路同步逻辑：同步更新node.model、manager.owner_model、manager.own_node.model、room_members中房主的model、UDP广播内容
- **相关文件**: `web_app.py`

### [P1-020] 加入房间时TCP连接被拒绝(WinError 10061)
- **描述**: 单机模式下ensure_cluster_initialized()初始化集群管理器后，完全没有启动ClusterServer监听30001端口，导致Worker节点连接127.0.0.1:30001时目标计算机积极拒绝
- **影响**: 单机模式下其他节点完全无法通过TCP连接加入房间
- **发现日期**: 2026-05-11
- **状态**: 🟢 已修复
- **修复内容**: 在单机模式懒加载初始化后自动调用manager.start_server()启动TCP服务器，同时给ClusterServer的socket添加2秒timeout确保可以正常停止
- **相关文件**: `evolution/cluster/connection.py`, `web_app.py`

## 💡 P3 - 低优先级（未来改进）

### [P1-010] discovery.py构造函数缺少host_port参数
- **描述**: ClusterDiscovery构造函数在初始化时没有host_port参数，但在connection.py中调用时传入了该参数
- **影响**: 初始化发现服务时会报错
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 在discovery.py的__init__中添加host_port参数
- **相关文件**: `evolution/cluster/discovery.py`

### [P1-011] 局域网房间发现缺少API端点
- **描述**: 局域网UDP广播房间发现功能有底层实现，但缺少对外REST API暴露给前端
- **影响**: 其他玄枢项目无法在页面上发现局域网中的房间
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 添加三个新端点: /api/discovery/start_scan, /api/discovery/rooms, /api/discovery/stop
- **相关文件**: `web_app.py`

### [P1-012] 能力评估默认返回0.5分
- **描述**: 云端API和本地Ollama模型大部分不在硬编码排行榜里，直接返回默认0.5分
- **影响**: 用户所有模型都是0.5分，能力评估形同虚设
- **发现日期**: 2026-05-09
- **状态**: 🟢 已修复
- **修复内容**: 
  1. 大幅扩展模型排行榜，加入几乎所有主流模型
  2. 新增智能子串匹配（优先最长匹配）
  3. 新增基于参数量的自动估算规则（70b=0.90, 32b=0.85...）
  4. 云端API智能默认0.85，本地未知模型0.70
- **相关文件**: 
  - `evolution/cluster/capability.py`
  - `evolution/cluster/connection.py`

### [P3-001] 集群发现机制依赖局域网广播
- **描述**: 当前使用UDP广播发现节点，不支持跨网络
- **改进方向**: 支持手动指定节点地址
- **状态**: 🟢 已部分修复，新增扫描API
- **相关文件**: `evolution/cluster/discovery.py`

### [P3-002] 向量记忆功能需要手动启用
- **描述**: `USE_VECTOR_MEMORY` 默认关闭，需要手动安装依赖
- **改进方向**: 自动化检测和配置
- **状态**: 🔴 待处理
- **相关文件**: `config.py`, `memory_enhanced.py`
