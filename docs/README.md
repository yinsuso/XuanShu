## 🎯 快速导航

### 新人入门
1. 先读 [架构蓝图.md](./架构蓝图.md) 了解整体
2. 再看 [项目地图.md](./项目地图.md) 定位模块
3. 有疑问查 [接口清单.md](./接口清单.md)

### 开发中
1. 修改代码前先看 [项目地图.md](./项目地图.md) 定位
2. 遇到问题查 [技术债清单.md](./技术债清单.md)
3. 做出重要决策记录到 [决策日志.md](./决策日志.md)

### 复盘时
1. 使用 [复盘机制.md](./复盘机制.md) 中的模板
2. 将复盘文档放到 `复盘/` 目录
3. 更新 [技术债清单.md](./技术债清单.md)

## 🔑 核心原则

> 所谓"上善若水，水善利万物而不争"。
> 文档如水，润物无声，用时取之。

1. **及时更新** - 代码变更后尽快更新对应文档
2. **精准简洁** - 文档要清晰准确，不冗余
3. **可追溯** - 记录变更历史，便于追查
