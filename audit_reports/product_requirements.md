## 一、项目概述

### 1.1 项目定位

**Local-Agent** 是一个基于 Hermes Agent 设计理念的本地化 AI Agent 框架，核心特点是：

- **内网闭环**：所有数据在本地处理，不依赖云端服务
- **极简设计**：MVP 理念，由简入繁
- **自我进化**：具备复盘、技能生成能力
- **多模式支持**：简单对话、任务规划、ReAct 循环、多 Agent 协作

### 1.2 核心价值主张

| 维度 | 价值描述 |
|------|----------|
| **隐私保护** | 数据完全本地化，适合敏感场景 |
| **低成本** | 支持 Ollama 本地模型，无需 API 费用 |
| **可扩展** | 技能系统支持自定义扩展 |
| **智能化** | ReAct 思考、多 Agent 协作提升任务完成质量 |

## 三、用户场景分析

### 3.1 典型用户画像

| 用户类型 | 需求 | 使用场景 |
|----------|------|----------|
| **开发者** | 代码辅助、文件操作 | 本地代码审查、文件批量处理 |
| **研究员** | 数据分析、文档生成 | JSON 数据分析、报告生成 |
| **隐私敏感用户** | 数据本地化 | 处理敏感文档、内部系统 |
| **AI 爱好者** | 探索 Agent 能力 | 学习 Agent 架构、自定义技能 |

### 3.2 核心用户场景

#### 场景 1：代码辅助开发
```
用户输入："帮我分析 /www/wwwroot/local-agent/agent.py 的代码结构"
期望流程:
1. Agent 调用 list_python_files 获取文件列表
2. 调用 read_file 读取目标文件
3. 调用 analyze_python_file 进行代码分析
4. 输出结构化分析报告
```

#### 场景 2：文件批量处理
```
用户输入："统计 data/ 目录下所有 JSON 文件的内容"
期望流程:
1. 调用 list_dir 获取文件列表
2. 循环调用 read_json 读取每个文件
3. 调用 analyze_json_structure 分析结构
4. 汇总输出统计结果
```

#### 场景 3：复杂任务规划
```
用户输入："帮我创建一个新项目，包含 README、requirements.txt 和基础代码结构"
期望流程:
1. Planner 拆解任务为子任务
2. Executor 依次执行文件创建
3. Reviewer 验证结果
4. Integrator 输出最终报告
```
**现状**：多 Agent 协作模块已实现但未集成到主流程。

## 四、缺失功能与待优化点

### 4.1 核心功能缺失

| 优先级 | 功能 | 影响 | 建议 |
|--------|------|------|------|
| **P0** | 执行模式自动选择 | 用户无法体验高级功能 | 实现 `auto` 模式，根据任务复杂度选择策略 |
| **P0** | ReAct 循环集成 | 失去深度思考能力 | 将 `react_loop.py` 集成到 `agent.py` |
| **P1** | 多 Agent 协作集成 | 复杂任务效率低 | 将 `multi_agent.py` 集成到主流程 |
| **P1** | 工作流引擎集成 | 无法使用预设模板 | 将 `workflow.py` 集成到主流程 |
| **P2** | 技能管理面板 | Web 界面功能不完整 | 在 Web 界面添加技能管理功能 |

### 4.2 用户体验优化

| 优先级 | 优化点 | 说明 |
|--------|--------|------|
| **P0** | 错误处理优化 | 当前错误信息不够友好，需增强 |
| **P1** | 技能调用确认 | 危险操作需用户确认（当前自动通过） |
| **P1** | 进度反馈 | 长任务需实时进度显示 |
| **P2** | 上下文清理 | 对话历史过长时自动清理 |

### 4.3 技术债务

| 问题 | 影响 | 建议 |
|------|------|------|
| **未使用的代码文件** | 维护成本高 | 删除或归档旧版本（`agent.py`, `agent_v3_1.py`, `agent_skill.py`） |
| **技能系统重复实现** | 代码冗余 | 统一 `skills/base.py` 和 `skills/__init__.py` 两套系统 |
| **Import 不规范** | 可读性差 | 统一使用绝对导入，避免循环依赖 |
| **缺少单元测试** | 质量风险 | 添加核心模块测试 |
